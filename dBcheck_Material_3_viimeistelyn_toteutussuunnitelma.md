# dBcheck Material 3 -viimeistelyn toteutussuunnitelma

**Goal:** Toteuttaa `dBcheck_M3_refinement_plan.md`-dokumentin mukainen Material 3 -viimeistely ilman tuotevirtojen, reittien, ViewModel-dataflow’n tai mittauslogiikan muutoksia.

**Architecture:** Ensin keskitetään tokenit ja jaetut komponentit, jotta pintahierarkia, spacing, lukitut tilat, chart-tyyli ja status-tilat ovat yhden lähteen takana. Sen jälkeen ruudut viimeistellään vaiheittain Meter → Analytics/History/Detail → Settings/setup → Camera/external surfaces. Toteutus on visuaalinen refaktorointi; uudet jaetut komponentit dokumentoidaan `AGENTS.md`- ja `memory/MEMORY.md`-tiedostoihin, koska vastuita keskitetään.

**Viralliset lähteet tarkistettu 2026-07-09:** [Compose Material 3](https://developer.android.com/develop/ui/compose/designsystems/material3), [Compose Material3 release notes](https://developer.android.com/jetpack/androidx/releases/compose-material3), [Compose accessibility API defaults](https://developer.android.com/develop/ui/compose/accessibility/api-defaults), [Material 3 components](https://m3.material.io/components).

## Testaus ja hyväksyntä

- Aja jokaisen vaiheen jälkeen vähintään kohdistetut testit: komponenttimuutoksille `.\gradlew.bat :app:testDebugUnitTest --tests "com.dbcheck.app.ui.*"`; meter/analytics/history-vaiheille lisäksi nykyiset geometry/source-contract-testit.
- Ennen yhdistämistä aja: `.\gradlew.bat :app:testDebugUnitTest :app:lintDebug :app:detektDebug :app:debugStabilityCheck :app:validateDebugScreenshotTest :app:assembleDebug`.
- Screenshot-baseline päivitetään vain visuaalisen hyväksynnän jälkeen komennolla `.\gradlew.bat :app:updateDebugScreenshotTest`; tämän jälkeen `.\gradlew.bat :app:validateDebugScreenshotTest` on pakollinen.
- Manuaalinen UAT-matriisi: 360×592 compact height, 360×740 phone, ≥600dp navigation rail, dark/light, fontScale 1.3 ja 1.5, Meter idle/recording/dosimeter/free locked, Analytics overview/spectral/environment locked+unlocked, History empty/search/session list, Session Detail locked/export/histogram, Settings Pro/free, camera granted/denied/unavailable, hearing setup/active/results.
- Hyväksymiskriteerit: yksi hero per screen, default cardit `surfaceContainer`, sankari/live/tappable-kortit eksplisiittisesti `Elevated`, 12dp in-group ja 32dp section gaps, ei uusia tuoteominaisuuksia, ei reitti- tai ViewModel-dataflow-muutoksia, ei uusia riippuvuuksia, 48dp touch targetit säilyvät, chart contentDescription säilyy tai paranee, warning/error-väri esiintyy vain semanttisena signaalina.

## Oletukset

- Toteutus kattaa koko dokumentin, ei vain Priority 1 -kohtia.
- `UI-SPEC.md` on nykytilan lähde; sitä ei päivitetä tavoitetilaksi ellei erikseen pyydetä.
- `lc` ja `sc` ovat käyttäjän ajamia wrapper-skriptejä, joten toteutus käyttää Gradle-testejä ja -checkejä suoraan.
- Riippuvuuksia tai version catalogia ei muuteta; Material3 expressive slider -päivitystä ei tehdä, vaan nykyinen slider säilyy tokenisoituna.
